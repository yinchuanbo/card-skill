## 概览

借助**性能**面板，您可以记录 Web 应用的 CPU 性能配置文件。分析配置文件，找出潜在的性能瓶颈以及优化资源使用方式。

您可以使用**效果**面板执行以下操作：

- 记录性能配置文件。
- 更改截取设置。
- 分析效果报告。

如需有关提升网站性能的全面指南，请参阅[分析运行时性能](https://developer.chrome.com/docs/devtools/performance?hl=zh-cn)。

## 打开“效果”面板

如需打开**性能**面板，请[打开开发者工具](https://developer.chrome.com/docs/devtools/open?hl=zh-cn)，然后从顶部的一系列标签页中选择**性能**。

或者，您也可以按照以下步骤使用**命令菜单**打开**效果**面板：

1.  [打开开发者工具](https://developer.chrome.com/docs/devtools/open?hl=zh-cn)。
2.  按以下任一键打开**命令菜单**：
3.  macOS：Command+Shift+P
4.  Windows、Linux、ChromeOS：Ctrl+Shift+P ![包含 ](https://developer.chrome.com/static/docs/devtools/performance/overview/image/cmdm-Performance-panel.png?hl=zh-cn)
5.  开始输入 `Performance panel`，选择**显示“效果”面板**，然后按 Enter 键。

## 实时观察 Core Web Vitals

当您打开**效果**面板时，该面板会立即捕获并显示您的本地 [Largest Contentful Paint (LCP)](https://web.dev/articles/lcp?hl=zh-cn) 和 [Cumulative Layout Shift (CLS)](https://web.dev/articles/cls?hl=zh-cn) 指标，并告知您其得分（良好、需要改进或欠佳）。

如果您与网页互动，**效果**面板还会捕获本地 [Interaction to Next Paint (INP)](https://web.dev/articles/inp?hl=zh-cn) 及其得分。除了 LCP 和 CLS 之外，您还可以通过该得分全面了解网页在您的网络连接和设备上的 [Core Web Vitals](https://web.dev/articles/vitals?hl=zh-cn)。

在**互动**和**布局偏移**标签页中的三个指标卡片下，您可以找到包含捕获的互动和布局偏移信息的表格，包括元素、时间、阶段（对于互动）和得分（对于布局偏移）。如要清除这两个列表，请依次点击 **清除**。

如需查看指标得分的细分数据，请将鼠标悬停在指标值上，以查看提示。

### 将您的体验与用户的体验进行比较

您还可以从 [Chrome 用户体验报告](https://developer.chrome.com/docs/crux?hl=zh-cn)中提取现场数据，并将您网站用户的体验与本地指标进行比较。

如需添加字段数据，请执行以下操作：

1.  依次点击**效果** > **后续措施** > **现场数据**，然后点击**设置**。

    ![“后续步骤”部分中的“设置”按钮。](https://developer.chrome.com/static/docs/devtools/performance/overview/image/field-data-setup.png?hl=zh-cn)

2.  在**配置字段数据提取**对话框中，记下**隐私权披露声明**，然后点击**确定**。

    #### 高级：在开发环境和生产环境之间设置映射...

    设置好现场数据提取后，**效果**面板现在会显示您的本地指标得分与用户体验的指标得分之间的对比情况。您可以在右侧的**字段数据**部分中查看收集期。

    ![字段数据提取后的收集周期。](https://developer.chrome.com/static/docs/devtools/performance/overview/image/field-data-fetched.png?hl=zh-cn)

    如需查看指标得分的细分数据，请将鼠标悬停在指标值上，以查看提示。

### 配置您的环境，使其更符合用户的环境

按照上一部分所述设置字段数据提取后，**效果**面板会为您提供有关如何配置环境以更好地契合用户体验的建议。

如需配置您的环境，请执行以下操作：

1.  在每个指标卡片中，展开**考虑本地测试条件**部分（如果有），然后阅读建议。

    ![每个指标卡片中展开的“考虑本地测试条件”部分。](https://developer.chrome.com/static/docs/devtools/performance/overview/image/env-recs.png?hl=zh-cn)

    在本例中，为了更好地匹配用户体验，您可能需要使用常见的桌面屏幕尺寸，并降低 CPU 和网络速度。

2.  如需与此示例的环境配置保持一致，请执行以下操作：

    1.  将视口设置为常见的屏幕尺寸之一（例如 720p 或 1080p）。如需模拟特定设备和屏幕尺寸，您可以使用**元素**面板中的[设备模式](https://developer.chrome.com/docs/devtools/device-mode?hl=zh-cn)。
    2.  在本例中，该网站的 82% 用户使用桌面设备进行浏览。为确保您将本地指标得分与正确的现场数据进行比较，您可以依次选择**现场数据** > **设备**下拉列表中的**桌面设备**。
    3.  在**环境设置**部分，将**网络**下拉列表设置为“快速 4G”，将**CPU** 设置为“20 倍降速”。您还可以在同一部分 **停用网络缓存**。

3.  配置好环境后，请重新加载页面，与其互动以捕获本地 INP，然后再次比较指标得分。

    ![环境配置为与真实用户体验相匹配。](https://developer.chrome.com/static/docs/devtools/performance/overview/image/env-config.png?hl=zh-cn)

    现在，指标得分似乎更接近用户的实际体验。因此，**考虑本地测试条件**部分已从指标卡片中移除。

现在，您可以开始改进网站的[核心 Web 指标](https://web.dev/articles/vitals?hl=zh-cn)了：

- [优化 LCP](https://web.dev/articles/optimize-lcp?hl=zh-cn)
- [优化 INP](https://web.dev/articles/optimize-inp?hl=zh-cn)
- [优化 CLS](https://web.dev/articles/optimize-cls?hl=zh-cn)

### 更改截取设置

借助**捕获设置**，您可以更改 DevTools 捕获性能录制内容的方式，并在报告中提供更多信息。点击**拍摄设置** ，以访问**拍摄设置**菜单。

从**拍摄设置**菜单中选择以下选项：

- [**停用 JavaScript 示例**](https://developer.chrome.com/docs/devtools/performance/reference?hl=zh-cn#disable-js-samples)：停用在录制期间调用并显示在**主要**轨道中的 JavaScript 调用堆栈的录制。会降低性能开销。
- [**启用高级绘制插桩（慢速）**](https://developer.chrome.com/docs/devtools/performance/reference?hl=zh-cn#paint)：捕获高级绘制插桩。严重影响性能。
- [**启用 CSS 选择器统计信息（较慢）**](https://developer.chrome.com/blog/new-in-devtools-125?hl=zh-cn#perf)：捕获 CSS 选择器统计信息。严重影响性能。
- [**CPU 节流**](https://developer.chrome.com/docs/devtools/performance/reference?hl=zh-cn#cpu-throttle)：模拟较慢的 CPU 速度。
- [**网络节流**](https://developer.chrome.com/docs/devtools/performance/reference?hl=zh-cn#network-throttle)：模拟较慢的网络速度。
